================================================================================
PROBLEM ANALYSIS
================================================================================

CURRENT CHALLENGES:

For Students:
• Manual, time-consuming application processes
• Limited visibility into available opportunities
• Corruption and favoritism in placements
• Skills-opportunity mismatch
• Complex documentation requirements (letters, NITA forms, insurance, etc.)

For Institutions:
• No centralized tracking of student placements
• Manual processing overhead
• Limited placement success data
• Difficulty monitoring student progress

For Companies:
• Time-consuming manual screening
• Difficulty finding qualified candidates
• Administrative burden
• Lack of structured communication

================================================================================
PROPOSED SOLUTION
================================================================================

An intelligent, automated three-sided platform that:

1. AUTOMATES matching between students and companies using AI/ML
2. ELIMINATES manual processes through document auto-generation
3. ENSURES fairness by removing human bias from initial matching
4. PROVIDES transparency through real-time tracking
5. INTEGRATES with existing institutional systems
6. COMPLIES with NITA regulatory requirements

KEY BUSINESS LOGIC CLARIFICATIONS
================================================================================

1. MULTI-OFFER RESOLUTION:
   - Student receives multiple acceptances
   - System auto-ranks by: match score (40%), stipend (30%), location (20%), rating (10%)
   - Auto-selects top offer after 48-hour timeout
   - Student can manually choose or appeal within 24 hours
   - Rejected companies notified with reason

2. FINANCIAL MATCHING:
   - Students set "requires_paid_placement" flag
   - If true, only matched with stipend-offering opportunities
   - Minimum stipend requirement honored
   - M-Pesa integration for payment tracking

3. AUTOMATED APPLICATIONS:
   - Students enable auto-apply feature
   - System auto-submits to opportunities with match score > 80
   - Max applications per day configurable
   - Student can cancel within 24 hours
   - All documents auto-generated

4. FEEDBACK LOOP:
   - Students rate placements (1-5 stars)
   - Feedback used to retrain ML models monthly
   - Match accuracy improves over time
   - A/B testing for algorithm improvements

5. TRANSPARENCY:
   - Every decision logged in audit trail

## 🔗 Key Business Logic Clarified

### 1. Multi-Offer Resolution Process

```
Student receives offers from Company A, B, C
↓
System ranks:
  - Company A: Score 85, Stipend 15K, Location Match, Rating 4.5
  - Company B: Score 82, Stipend 20K, Location Mismatch, Rating 4.2
  - Company C: Score 88, Stipend 10K, Location Match, Rating 4.8
↓
Ranking calculation:
  - A: (85×0.4) + (20K×0.3) + (100×0.2) + (4.5×10×0.1) = 74.5
  - B: (82×0.4) + (20K×0.3) + (0×0.2) + (4.2×10×0.1) = 71.0
  - C: (88×0.4) + (10K×0.3) + (100×0.2) + (4.8×10×0.1) = 74.0
↓
Company A auto-selected (highest score)
↓
Student notified: "We selected Company A. Appeal within 24h"
Companies B & C notified: "Student accepted another offer"
```

```
New opportunity posted: Software Dev at Company X
↓
Matching algorithm runs
↓
Student Y has match score: 87/100

Check student settings:
  - auto_apply_enabled: true
  - max_auto_applications_per_day: 5 (used: 2)
  - min_auto_match_score: 80
  - Company X not blacklisted: ✓
↓
Auto-generate documents:
  - Application letter

Submit application automatically
↓
Notify student: "We applied to Company X for you! (87% match)"
↓
Student has 24 hours to cancel
```
